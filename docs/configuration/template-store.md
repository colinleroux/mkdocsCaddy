
# Template Store

Templates are central to the Docmosis document generation process. Each document generated by Tornado is based on a template that defines the layout, formatting, and document structure.

The **Template Store** defines where Tornado will locate these templates.

Templates can be stored in several locations depending on the needs of the deployment.

Common options include:

* a local directory on the Tornado server
* a shared network location
* cloud storage services such as object storage buckets

The simplest configuration is a local directory.

Example configuration:

```
templatesDir=/opt/docmosis/templates
```

In container environments this may instead refer to a mounted volume:

```
DOCMOSIS_TEMPLATESDIR=/docmosis-template-store
```

When using a local template store, the directory should contain the document templates used by the application. These are typically Microsoft Word (`.docx`) or OpenDocument (`.odt`) templates designed using the Docmosis template syntax.

Templates can be organised into subdirectories if required, allowing applications to reference templates using structured paths.

For example:

```
templates/
   invoices/
   contracts/
   reports/
```

Maintaining a clear and consistent template structure can simplify application integration and make template management easier as systems grow.
