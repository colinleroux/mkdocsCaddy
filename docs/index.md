# Welcome to the Tornado Documentation Project

## Clearer, practical documentation for Docmosis Tornado

This documentation site has been created to support the practical use and operation of Docmosis Tornado.

Its aim is simple: **clear the fog and reduce the noise**.

Rather than repeating the full official documentation, this site focuses on making important topics easier to understand through clearer structure, operational context, and practical examples.

## Purpose

This site is intended to help users and support teams make better decisions about how Tornado is deployed, configured, monitored, and maintained.

It is especially focused on:

- deployment planning
- configuration clarity
- operational guidance
- troubleshooting
- common support tasks
- practical reference material

## Navigation

The site is organised into focused sections:

- **Understanding what Tornado is**
- **Configuration**
- **Monitoring & Logging**
- **Troubleshooting**
- **How-To Guides**
- **Advanced Topics**
- **Reference**
- **FAQ**

Each section is designed to group related information together so that it is easier to locate and use.

## Relationship to official documentation

This documentation is **not a replacement for official Docmosis documentation**.

Instead, it should be used alongside the official product documentation. The official guides remain the authoritative source for installation instructions, formal product detail, and version-specific reference material.

This site complements that material by presenting key operational topics in a way that is easier to navigate, easier to interpret, and more directly useful in day-to-day work.

## Suggested starting points

- New to Tornado? Start with **Planning**
- Working on setup? Go to **Configuration**
- Investigating a problem? Use **Troubleshooting**
- Looking for logs or diagnostics? See **Monitoring & Logging**
- Trying to complete a task? Open **How-To Guides**

## Final note

This is a working documentation project intended to improve clarity over time. Pages will continue to evolve as useful examples, support insights, and operational guidance are added.
